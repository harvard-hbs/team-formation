  4# title: Team Formation as Constraint Satisfaction
  5# ---
  6# ## Problem Statement
  7#
  8# Dividing a large learning cohort into smaller teams for group work,
  9# discussion, or other activity is a common requirement in many learning
 10# contexts. It is easy to automate the formation of randomly assigned
 11# teams, but there can be rules, guidelines, and goals guiding the
 12# desired team composition to support learning objectives and other
 13# goals which can complicate manual and automated team creation.
 14#
 15# The approach described in this document provides a technical framework
 16# and implementation to support specifying team formation objectives in
 17# a declarative fashion and can automatically generate teams based on
 18# those objectives. There is also a description of how to measure and
 19# evaluate the created teams with respect to the specified objectives.
 20#
 21# The team formation objectives currently supported are team size and
 22# *diversification* and *clustering* around participant
 23# attributes. *Diversification* in this context is defined as the goal
 24# of having the distribution of a particular attribute value on each
 25# team reflect the distribution of that attribute value in the overall
 26# learning cohort. For example, if the overall learning cohort has 60%
 27# women and 40% men, a diversification goal on gender would attempt to
 28# achieve 60/40 female/male percentages on each team or, more
 29# specifically, to achieve the female/male participant counts that are
 30# closest to 60%/40% for the particular team size.
 31#
 32# *Clustering* is defined as the goal of having all team members share a
 33# particular attribute value. For example, if there is a `job_function`
 34# attribute with values of `Contributor`, `Manager`, and `Executive` a
 35# clustering goal would be to have each team contain participants with a
 36# single value of the `job_function` attribute to facilitate sharing
 37# of common experiences.
 38#
 39# Cluster variables can also be multi-valued indicated by a list of
 40# acceptable values for the participant. For example, if there is a
 41# `working_time` variable with hour ranges `00-05`, `05-10`, `10-15`,
 42# `15-20`, and `20-24`. A participant might have the values `["00-05",
 43# "20-24"]` indicating that both those time ranges are acceptable.
 44#
 45# In order to balance possibly conflicting objectives and goals of the
 46# team formation process we allow a weight to specified for each
 47# constraint to indicate the priority of the objective in relation
 48# to the others.
 49#
 50# ## Team Formation as Constraint Satisfaction using CP-SAT
 51#
 52# The problem of dividing participants into specified team sizes guided
 53# by diversity and clustering constraints can be stated as a [Constraint
 54# Satisfaction
 55# Problem](https://en.wikipedia.org/wiki/Constraint_satisfaction_problem)
 56# (CSP) with a set of variables with integer domains and constraints on
 57# the allowed combinations.
 58#
 59# There is a very efficient constraint solver that uses a variety of
 60# constraint solving techniques from the Google Operational Research
 61# team called [Google OR-Tools
 62# CP-SAT](https://developers.google.com/optimization/cp/cp_solver) that
 63# we are using for this team assignment problem.
 64#
 65# The remainder of the document describes how to frame the team
 66# formation problem in the CP-SAT constraint model to be solved by the
 67# CP-SAT solver.
 68#
 69# ## Input Data
 70#
 71# The input to the team formation process is a set of participants with
 72# category-valued attributes, a target team size, and a set of
 73# constraints. The specification of the constraints is done with a
 74# dictionary with keys attribute names from the `participants` data frame as
 75# keys, a type of `diversify` or `cluster`, and a numeric `weight`.

151    def __init__(
152        self, participants, constraints, target_team_size, less_than_target=False
153    ):
154        
155        self.model = cp_model.CpModel()
156        """The CP-SAT model for creating variables and constraints"""
157
158        self.solution_found = False
159        """The `solve` method will set this to `True` if a solution is found"""
160        
161        self.participants = participants
162        """Input variable: participants with attributes"""
163
164        self.num_participants = len(self.participants)
165        """Number of participants"""
166
167        self.attr_constraints = constraints.set_index("attribute").to_dict("index")
168        """Input variable: constraints with name, type, and priority"""
169        
170        for attr_name in self.attr_constraints:
171            ct_type = self.attr_constraints[attr_name]["type"]
172            if ct_type not in self.CONSTRAINT_TYPES:
173                raise ValueError("Unknown constraint type", ct_type)
174        self.attr_names = list(self.attr_constraints.keys())
175
176        #: Input variable: desired target team size
177        self.target_team_size = target_team_size
178
179        #: Input variable: make non-target-size teams one bigger or smaller
180        self.less_than_target = False
181
182        #: Calculated sizes of the teams
183        self.team_sizes = calc_team_sizes(
184            self.num_participants, self.target_team_size, self.less_than_target
185        )
186        #: Number of teams
187        self.num_teams = len(self.team_sizes)
188
189        # ### Attribute Value Boolean Variable Names
190        #
191        # Turn the categorical attribute values
192        # into a set of boolean variables names
193        self.attr_vals = {}
194        for attr_name in self.attr_names:
195            bool_vars = categories_to_bool_vars(attr_name, self.participants[attr_name])
196            self.attr_vals[attr_name] = bool_vars
197
198        # ### Participant Variables
199        #
200        # Next we create the `parti_vars` array with one entry for each
201        # participant. Each particpant's entry in the array contains a map from
202        # attribute name to the list of CP-SAT boolean variables indicating
203        # whether the participant has that value of the attribute. A constraint
204        # of `bool_var == 1` for the presence of the value in the data and
205        # `bool_var == 0` for the other attribute values. (Open question: should
206        # there be a constraint for `sum(bool_vars) == 1` or is it enough to
207        # have the specific `0` and `1` constraints on each variables?)
208
209        self.parti_vars = []
210        for id, parti in self.participants.iterrows():
211            attr_vars = defaultdict(list)
212            for attr_name in self.attr_vals:
213                bool_vars = []
214                parti_vals = parti[attr_name]
215                if isinstance(parti_vals, list):
216                    verb = "has"
217                else:
218                    verb = "is"
219                    parti_vals = [parti_vals]
220                attr_vals = [
221                    make_attr_value_name(attr_name, pv, verb) for pv in parti_vals
222                ]
223                for bool_var_cat in self.attr_vals[attr_name]:
224                    bool_var = self.model.NewBoolVar(bool_var_cat)
225                    bool_vars.append(bool_var)
226                    if bool_var_cat in attr_vals:
227                        self.model.Add(bool_var == 1)
228                    else:
229                        self.model.Add(bool_var == 0)
230                attr_vars[attr_name] = bool_vars
231            self.parti_vars.append(attr_vars)
232
233        # ### Team Assignment Constraints
234        #
235        # We have already calculated the size of each team reflected in the
236        # `team_sizes` variable that is indexed by team number and holds the
237        # size of each team. We now create an array of `num_teams` boolean
238        # variables for each participant indicating whether that participant is
239        # on the corresponding team. That array of team assignment variables is
240        # put in the `parti_vars` map of each participant with the key `"team"`.
241        #
242        # We also add a constraint that the sum of each participant's team
243        # boolean variables must be `1` (a participant can only be on one team)
244        # and another constraint that the sum of each of the boolean variables for
245        # each team must be equal to the team size to ensure the correct number
246        # of participants in each team.
247
248        ## Create team variables
249        for id in range(self.num_participants):
250            # Each participant has a boolean per team to indicate team membership
251            parti_team_vars = []
252            for team_num, team_size in enumerate(self.team_sizes):
253                team_var = self.model.NewBoolVar(f"parti_{id}_in_team_{team_num}")
254                parti_team_vars.append(team_var)
255            self.parti_vars[id]["team"] = parti_team_vars
256            # Add a constraint so a participant can only be in one team
257            self.model.Add(cp_model.LinearExpr.Sum(parti_team_vars) == 1)
258
259        ## Enforce desired team size on each team
260        for team_num, team_size in enumerate(self.team_sizes):
261            team_vars = [
262                self.parti_vars[id]["team"][team_num]
263                for id in range(self.num_participants)
264            ]
265            self.model.Add(cp_model.LinearExpr.Sum(team_vars) == team_size)
266
267        # ### Team Attribute Value Count Variables
268        #
269        # Both the diversity and clustering constraints used for team formation
270        # are defined in terms of team-by-team attribute value counts. We use a
271        # `team_vars` map keyed by the attribute name with an array value
272        # indexed by team; for each team there is an array indexed by attribute
273        # value containing an integer variable representing the count of
274        # that attribute value in the corresponding team.
275        #
276        # [Describe the mechanism for calculating the counts here.]
277        #
278        # The count of the number of females in team `0` is
279        # `team_vars["gender"][0][0]`.
280
281        ## Create the map from attribute names to array of teams
282        ## containing attribute value counts.
283        self.team_vars = {
284            attr_name: self.create_attr_counts(attr_name)
285            for attr_name in self.attr_names
286        }
287
288        # ### Team Diversity and Clustering Constraints
289        #
290        # While all of the constraints defined so far have been all or nothing
291        # and necessary for the proper functioning of the model (for example,
292        # the setting of the participant attribute values and team sizes) the
293        # diversity and clustering constraints will be defined with cost
294        # functions that go up when the teams are farther away from the desired
295        # state and equal to zero in the optimal situation. This allows for
296        # over-constraining the model and letting the constraint engine take
297        # into account the weights, providing a solution as close as we can get
298        # given the participants and their provided attributes.
299        #
300        # #### Diversity
301        #
302        # We consider a team to be ideally diverse with respect to a particular
303        # attribute when the distribution of each attribute value is the same in
304        # the team as it is in the entire participant population. For example if
305        # there are 60% women and 40% men in the full population, then a team
306        # will be ideally diverse for the gender attribute if there are 60%
307        # women and 40% men in that team. We turn those percentage values into
308        # integer count targets for each attribute value in a particular-sized
309        # team and define the cost function as the absolute value of the
310        # difference from the target count, summed across all the variables and
311        # multipled by the constraint weight.
312        #
313        # We define a `population_distribution` method that returns the
314        # percentage of each attribute in the full population and then cache
315        # that in a `population_dist` map for later use.
316        #
317        # The `value_count_targets` method uses the population distribution and
318        # a team size to return the ideal count for each attribute value in a
319        # team of a particular team size. It is important that rounding be used
320        # when turning the percentage into a target value count (rather than
321        # truncation) to ensure that the target counts add up to the team size.
322
323        ## Cache attribute value distributions for diversity constraints.
324        self.population_dist = {}
325        for attr_name in self.attr_constraints:
326            constraint = self.attr_constraints[attr_name]
327            if constraint["type"] == self.CT_DIVERSIFY:
328                self.population_dist[attr_name] = self.population_distribution(
329                    attr_name
330                )
331
332        # #### Clustering
333        #
334        # We consider a team to be ideal with respect to an attribute clustering
335        # constraint if all of the members of the team have a single value for
336        # that attribute. For example, if the `job_function` attribute has
337        # values `Contributor`, `Manager`, and `Executive` and a team's size is `3`,
338        # then one ideal team configuration is to have all three team members
339        # with a `job_function` value of `Contributor`. One way to test for that
340        # situation is to compute the maximum of the team's attribute value
341        # counts and take its maximum. If the maximum is equal to the team size,
342        # then ideal clustering is achieved for the team. We define the
343        # clustering cost function to be the difference between the maximum of a
344        # team's attribute value counts as the team size, summed across all of
345        # the teams and multiplied by the constraint weight.
346        #
347        # [Put some more details about the calculation mechanism here.]
348        #
349        # The cost constraints variables are kept in a flat list containing all
350        # of the cost variables. We also calculate the population distributions
351        # for the diversity constraints in this step.
352        #
353
354        # Diversity constraints rely on a measurement of the distribution
355        # of the attribute values across the entire population.
356        self.population_dist = {}
357
358        # The list of all cost variables
359        self.attr_costs = []
360
361        for attr_name in self.attr_constraints:
362            constraint = self.attr_constraints[attr_name]
363            if constraint["type"] == self.CT_DIVERSIFY:
364                self.population_dist[attr_name] = self.population_distribution(
365                    attr_name
366                )
367                costs = self.create_diversity_costs(attr_name)
368            if constraint["type"] == self.CT_CLUSTER:
369                costs = self.create_clustering_costs(attr_name)
370            if constraint["weight"] != 1:
371                costs = [cost * constraint["weight"] for cost in costs]
372            self.attr_costs += costs
373
374        ## Minimize the sum of the cost variables
375        #
376        # The last step before solving for team assignments is informing the model
377        # of the need to minimize the sum of the cost variables.
378        #
379        self.model.Minimize(cp_model.LinearExpr.Sum(self.attr_costs))

394    def create_attr_counts(self, attr_name):
395        """Create attribute value counts for each team."""
396        team_attr_counts = []
397        attr_vals = self.attr_vals[attr_name]
398        for team_num, team_size in enumerate(self.team_sizes):
399            team_counts = []
400            for attr_val_index, attr_val in enumerate(attr_vals):
401                parti_counts = []
402                team_attr_count = self.model.NewIntVar(
403                    0, team_size, f"team_{team_num}_{attr_val}_count"
404                )
405                for id in range(self.num_participants):
406                    team_var = self.parti_vars[id]["team"][team_num]
407                    attr_var = self.parti_vars[id][attr_name][attr_val_index]
408                    parti_count = self.model.NewBoolVar(
409                        "parti_{id}_team_{team_num}_{attr_val}"
410                    )
411                    # Use team_var*attr_var to calculate And(team_var, attr_var)
412                    self.model.AddMultiplicationEquality(
413                        parti_count, [team_var, attr_var]
414                    )
415                    # This is an alternative that doesn't seem to perform as well
416                    # self.model.Add(parti_count == attr_var).OnlyEnforceIf(team_var)
417                    # self.model.Add(parti_count == 0).OnlyEnforceIf(team_var.Not())
418                    parti_counts.append(parti_count)
419                self.model.Add(team_attr_count == cp_model.LinearExpr.Sum(parti_counts))
420                team_counts.append(team_attr_count)
421            team_attr_counts.append(team_counts)
422        return team_attr_counts

433        pop_dist = self.population_dist[attr_name]

437    def create_diversity_costs(self, attr_name):
438        """Create costs variables for diversity optimization."""
439        diversity_costs = []
440        num_values = len(self.attr_vals[attr_name])
441        pop_dist = self.population_dist[attr_name]
442        for team_num, team_size in enumerate(self.team_sizes):
443            targets = self.value_count_targets(attr_name, team_size)
444            for val in range(num_values - 1):
445                cost_var = self.model.NewIntVar(
446                    0, team_size, f"{attr_name}_cost_{team_num}_{val}"
447                )
448                self.model.AddAbsEquality(
449                    cost_var,
450                    (self.team_vars[attr_name][team_num][val] - targets.iloc[val]),
451                )
452                diversity_costs.append(cost_var)
453        return diversity_costs

455    # #### Clustering
456    #
457    def create_clustering_costs(self, attr_name):
458        """Create cost variables for clustering optimization."""
459        clustering_costs = []
460        for team_num, team_size in enumerate(self.team_sizes):
461            max_count_var = self.model.NewIntVar(
462                0, team_size, f"{attr_name}_max_count[{team_num}]"
463            )
464            cost_var = self.model.NewIntVar(
465                0, team_size, f"{attr_name}_cost[{team_num}]"
466            )
467            self.model.AddMaxEquality(
468                max_count_var, self.team_vars[attr_name][team_num]
469            )
470            self.model.Add(cost_var == (team_size - max_count_var))
471            clustering_costs.append(cost_var)
472        return clustering_costs

474    # ## Solving the Model
475    #
476    # The `solve` method creates a `CpSolver` object and calls its `solve`
477    # method on the model we have created. The status is checked for an
478    # optimal or feasible solution and, if found, the team assignments
479    # are assigned to the participants.
480
481    def solve(self, solution_callback=None):
482        self.solver = cp_model.CpSolver()
483        # This alternative is for establishing a callback for interrupting before
484        # an optimal solution is found.
485        self.status = self.solver.Solve(self.model, solution_callback=solution_callback)
486        self.solution_found = (self.status == cp_model.OPTIMAL) or (
487            self.status == cp_model.FEASIBLE
488        )
489        if not self.solution_found:
490            print(f"Warning: solution was not found: {self.status}")
491        else:
492            team_assignments = []
493            for id in range(self.num_participants):
494                for team_num in range(self.num_teams):
495                    if self.solver.Value(self.parti_vars[id]["team"][team_num]):
496                        team_assignments.append(team_num)
497            self.participants["team_num"] = team_assignments

499    # ## Evaluating Team Assignments
500    #
501    # The primary way to evaluate the team assignments is to do a
502    # team-by-team calculation for each attribute constraint to
503    # determine how many participants have been mis-assigned
504    # in terms of diversifaction target count or membership in
505    # the predominant clustering group.
506
507    def evaluate_teams(self):
508        if not self.solution_found:
509            print("Warning: solution has not been found")
510            return
511        team_info_list = []
512        for team_num, team_size in enumerate(self.team_sizes):
513            for attr_name in self.attr_names:
514                ct_type = self.attr_constraints[attr_name]["type"]
515                if ct_type == self.CT_DIVERSIFY:
516                    pop_targets = self.value_count_targets(attr_name, team_size)
517                    team_partis = self.participants[
518                        self.participants["team_num"] == team_num
519                    ]
520                    team_counts = team_partis[attr_name].value_counts().sort_index()
521                    missed = (
522                        (pop_targets - team_counts)
523                        .fillna(0)
524                        # Only need of one pos/neg pair
525                        .pipe(lambda x: x[x > 0])
526                        .sum()
527                        .astype(int)
528                    )
529                elif ct_type == self.CT_CLUSTER:
530                    team_partis = self.participants[
531                        self.participants["team_num"] == team_num
532                    ]
533                    max_count = max_attr_value_count(team_partis[attr_name])
534                    missed = team_size - max_count
535                team_info = {
536                    "team_num": team_num,
537                    "team_size": team_size,
538                    "attr_name": attr_name,
539                    "type": ct_type,
540                    "missed": missed,
541                }
542                team_info_list.append(team_info)
543        team_info = pd.DataFrame(team_info_list)
544        return team_info

569def calc_team_sizes(pop_size, target_size, less_than=True):
570    if less_than:
571        num_teams = (pop_size + target_size - 1) // target_size
572        base_team_size = pop_size // num_teams
573        extra_members = pop_size % num_teams
574        team_sizes = [
575            (base_team_size + 1) if (i < extra_members) else base_team_size
576            for i in range(num_teams)
577        ]
578    else:
579        num_teams = pop_size // target_size
580        base_team_size = pop_size // num_teams
581        extra_members = pop_size % num_teams
582        team_sizes = [
583            base_team_size if (i >= extra_members) else (base_team_size + 1)
584            for i in range(num_teams)
585        ]
586    return team_sizes

485        # an optimal solution is found.
486        self.status = self.solver.Solve(self.model, solution_callback=solution_callback)

492        else:
493            team_assignments = []

495                for team_num in range(self.num_teams):

501    #
502    # The primary way to evaluate the team assignments is to do a
503    # team-by-team calculation for each attribute constraint to
504    # determine how many participants have been mis-assigned
505    # in terms of diversifaction target count or membership in
506    # the predominant clustering group.

508    def evaluate_teams(self):
509        if not self.solution_found:
510            print("Warning: solution has not been found")
511            return
512        team_info_list = []
513        for team_num, team_size in enumerate(self.team_sizes):
514            for attr_name in self.attr_names:
515                ct_type = self.attr_constraints[attr_name]["type"]
516                if ct_type == self.CT_DIVERSIFY:
517                    pop_targets = self.value_count_targets(attr_name, team_size)
518                    team_partis = self.participants[
519                        self.participants["team_num"] == team_num
520                    ]
521                    team_counts = team_partis[attr_name].value_counts().sort_index()
522                    missed = (
523                        (pop_targets - team_counts)
524                        .fillna(0)
525                        # Only need of one pos/neg pair
526                        .pipe(lambda x: x[x > 0])
527                        .sum()
528                        .astype(int)
529                    )
530                elif ct_type == self.CT_CLUSTER:
531                    team_partis = self.participants[
532                        self.participants["team_num"] == team_num
533                    ]
534                    max_count = max_attr_value_count(team_partis[attr_name])
535                    missed = team_size - max_count
536                team_info = {
537                    "team_num": team_num,
538                    "team_size": team_size,
539                    "attr_name": attr_name,
540                    "type": ct_type,
541                    "missed": missed,
542                }
543                team_info_list.append(team_info)
544        team_info = pd.DataFrame(team_info_list)
545        return team_info
